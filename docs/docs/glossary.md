# Glossary

Definitions of terms used throughout the Velora documentation.

---

### Account

A Solana account stores data on-chain. Velora uses two account types: `StreamConfig` (stream state) and `ClaimReceipt` (withdrawal records).

### Anchor

The Rust framework used to build the Velora program. Provides account serialization, instruction routing, PDA derivation, and CPI helpers.

### Authority

Permission to perform a specific action on a stream. Two kinds: **release authority** (triggers milestone releases) and **cancel authority** (cancels a stream).

### Bump

A single-byte value used in PDA derivation to find a valid off-curve address. Stored in `StreamConfig.vault_bump` so the program can sign CPI calls as the vault PDA.

### CPI (Cross-Program Invocation)

One Solana program calling another. Velora uses CPI to interact with the SPL Token program for token transfers (e.g., moving tokens from creator to vault, or vault to recipient).

### Claim Receipt

A `ClaimReceipt` PDA created on each withdrawal, recording the amount and timestamp. Enables off-chain indexing and idempotency checks.

### Cliff

A vesting type where all tokens unlock at a single timestamp. Nothing is claimable before the cliff.

### Creator

The address that creates a stream, funds the vault, and sets the vesting parameters. Usually a project treasury, team lead, or deployer.

### Devnet

Solana's development network. Use for testing before mainnet. Tokens have no real value.

### IDL (Interface Definition Language)

A JSON description of the program's instructions, accounts, and types. Generated by Anchor at build time. Used by the TypeScript SDK for type safety.

### InitSpace

An Anchor derive macro that computes the byte size of an account struct at compile time. Used for rent-exempt account allocation.

### Mint

The SPL Token mint address. Identifies which token a stream distributes (e.g., USDC, custom token).

### Milestone

A vesting type where tokens are unlocked manually by a designated release authority, not by time.

### MultiSig

A program (like Squads) that requires multiple signers to approve an action. Used as a release authority for committee-controlled milestone vesting.

### PDA (Program Derived Address)

An address derived from seeds and a program ID that has no corresponding private key. Only the program can sign for it. Used for `stream_config`, `vault`, and `claim_receipt`.

### Recipient

The address that receives unlocked tokens from a stream. Only the recipient can call `withdraw`.

### Rent

SOL required to keep an account alive on Solana. Accounts must be rent-exempt (hold minimum SOL balance). Velora accounts are allocated with rent exemption at creation.

### Reserved field

The `reserved: [u8; 30]` field in `StreamConfig`. Intentionally included in the space budget to allow future protocol upgrades without account reallocation.

### Schedule Type

The vesting algorithm: `Linear`, `Cliff`, `CliffLinear`, or `Milestone`. Determines how unlocked tokens are calculated.

### SPL Token

Solana's token standard. Velora operates on SPL Token accounts, distributing tokens according to vesting schedules.

### Stream

A Velora vesting instance. One stream = one `StreamConfig` account + one vault + zero or more `ClaimReceipt` accounts.

### Stream ID

A `u64` chosen by the creator when creating a stream. Scoped to the creator (not global). Used as a PDA seed for deterministic address derivation.

### Token Account

An SPL Token account that holds tokens. The vault is a PDA-owned token account that holds locked stream tokens.

### Vault

A PDA-owned token account that holds the locked tokens for a stream. Only the program can authorize transfers from the vault.

### Vesting

The process of releasing tokens over time or according to conditions. Velora implements four vesting types.

### Withdraw

The instruction that lets a recipient claim unlocked tokens from a stream. Creates a `ClaimReceipt` on each call.
